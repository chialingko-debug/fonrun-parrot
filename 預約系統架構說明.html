鳳潤鳥宿網站架構與預約功能說明

本說明依據 index.html 現有程式碼整理而成，作為日後維護或向他人說明的參考。

一、整體架構

這是一個純前端靜態網頁（單一檔案 index.html，內含 HTML + CSS + JavaScript），沒有獨立的後端伺服器或框架（不是 React/Vue，也沒有 Node.js 後端）。

  • 託管方式：放在 GitHub repository（chialingko-debug/fonrun-parrot），可透過 GitHub Pages 直接發布上線。
  • 字型：Google Fonts（Cormorant Garamond、Noto Serif TC、Zen Maru Gothic）＋部分標題使用系統內建標楷體。
  • 頁面結構：單頁式（One Page），用錨點（#pricing、#rules、#booking、#faq、#contact）做導覽捲動，沒有真正的多頁路由。
  • 互動邏輯：全部寫在 <script> 區塊內的原生 JavaScript（無框架），包含日曆元件、價格試算、表單驗證、分頁切換等。

二、預約功能的兩條資料路徑

網站的「線上預約」區塊（#booking）其實是兩個獨立系統串接起來的，分別負責「寫入」和「查詢」。

1. 送出預約（寫入）→ Formspree

  • 表單 id="bForm" 送出時觸發 submitForm(event)。
  • 送出前，JS 會把鳥寶資訊整理成文字摘要（鳥名、品種、是否需要灌奶/塗藥等），塞進隱藏欄位 birds_summary、estimated_total、price_breakdown。
  • 接著用 fetch() 把整個表單資料 POST 到第三方表單服務 Formspree： 
    https://formspree.io/f/mredzrzd
    Formspree 會把預約內容轉成 email 通知寄到你註冊時用的信箱。
  • 送出後（不論成功或失敗），前端都會隱藏表單、顯示「預約申請已送出」的成功畫面。
注意：目前程式碼中，Formspree 收到的資料不會自動寫入 Google Sheet 或 Google 日曆。也就是說「寫入」與「查詢」這兩條路徑，在程式碼層面沒有自動串接——除非你在 Formspree 後台另外設定了 Zapier／Make（Integromat）之類的自動化工具，把表單資料同步寫進 Google Sheet。這部分設定不在 index.html 裡，是在 Formspree 或 Zapier 後台，目前看不到。

2. 查詢預約 → Google Apps Script（GAS）

  • 查詢分頁呼叫 queryBooking()，用使用者輸入的電話號碼組成 GET 請求： 
    GAS_URL + '?action=query&phone=xxx&t=時間戳'
  • GAS_URL 是一個已部署的 Google Apps Script Web App 網址： 
    https://script.google.com/macros/s/AKfycbwUmd8yIErbEaKTTLQCXwIGQ_6_b7--ZmuPlPDHx0CbJYpOHJdLyS83JRqVbZTf3ssC/exec
  • 這個 Apps Script 後端（程式碼不在此專案資料夾中，是另外部署在你的 Google 帳號下的 .gs 腳本）負責：
    1. 讀取某個 Google Sheet 裡的預約資料列
    2. 依電話號碼篩選符合的紀錄
    3. 回傳 JSON 陣列，每筆包含姓名、鳥寶資訊、入退住日期時間、狀態（確認中／已確認／已取消）等欄位
  • 前端拿到 JSON 後直接渲染成查詢結果卡片。
Google 日曆部分：index.html 的前端程式碼中沒有任何地方直接呼叫 Google Calendar API。如果 Google Sheet 後面有串 Google 日曆（例如 Apps Script 用 CalendarApp 把預約寫入日曆事件），那段邏輯寫在 Apps Script（.gs）裡，不在這個專案、也不在 HTML 檔案中，目前無法查看。

三、目前架構的限制與需要補充確認的地方

  1. Formspree → Google Sheet 沒有自動串接的程式碼：除非在 Formspree／Zapier 後台另外設定，否則送出的預約不會自動進到查詢系統讀取的那份 Google Sheet。如果現在查詢功能能正常查到剛送出的預約，代表中間一定有別的自動化（Zapier/Make，或手動謄寫）。
  2. Apps Script 原始碼不在這個 repo：GAS_URL 指向的腳本內容（如何讀 Sheet、如何寫日曆）目前無法在此專案中查看或編輯，是獨立部署在 Google 帳號裡的。若要修改查詢邏輯或日曆串接，需要到 script.google.com 開啟對應的 Apps Script 專案。
  3. 若想要更完整、準確的串接說明（特別是 Sheet ↔ 日曆 ↔ Formspree 三者怎麼接起來），建議把 Apps Script 的 .gs 原始碼一併提供，可以一起補上完整流程圖。
